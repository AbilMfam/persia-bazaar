# DigiMall Laravel API — DirectAdmin shared hosting deployment

Targets **DirectAdmin-style** setups: LiteSpeed/OpenLiteSpeed or Apache behind **SSL termination**, **`public/` document root**, and **PHP + MySQL** managed from the panel.

---

## Target production host: `diigiimall.ir`

| Item | Value |
|------|--------|
| Canonical URL | **`https://diigiimall.ir`** (`APP_URL` + front-end `VITE_API_BASE_URL` base) |
| API base | **`https://diigiimall.ir/api`** *(subfolder under the same hostname)* |
| Deploy path | Upload the Laravel tree **`backend/`** to the hosting account directory you control; **set the domain Document Root** to **`.../backend/public`** (often mapped as `public_html` pointing at Laravel `public`). |
| DB | MySQL from DirectAdmin: see **`backend/.env.production.example`** (`DB_DATABASE`, `DB_USERNAME`, `local` vs `localhost` per provider). |

**Security:** `.env.production.example` carries **example** secrets for handoff convenience. Before / after deployment, rotate the DB password from the hosting panel **if anyone else ever had repo access.**

**Shared-hosting Laravel 11 gotcha:** The framework ships with default cache/session drivers targeting **database** unless config is published (see **`config/cache.php`** and **`config/session.php`** in this repo — defaults are **`file`** so `php artisan optimize:clear`, `cache:clear`, etc. succeed **before** migrations create `cache`/`sessions` tables).

## Final deploy commands (on the server after `backend/.env` exists)

Replace `/path/to/backend` with the real SSH path shown in hosting docs.

```bash
cd /path/to/backend

php artisan migrate --force

php artisan optimize:clear
php artisan config:clear
php artisan cache:clear

php artisan storage:link --force || true

# Optional but recommended once stable:
# php artisan config:cache && php artisan route:cache
```

Smoke test: **`curl -sS https://diigiimall.ir/api/health`**

## Deployment checklist

1. **Domain / document root**

   Point the subdomain (recommended: `api.yourdomain.com`) **document root** to Laravel’s **`public/`** folder (often `domains/api.yourdomain.com/public_html` after upload, or symlink).  
   If the host forbids nesting, see [“Document root workaround”](#document-root-not-on-public).

2. **PHP**

   MultiPHP selector: **PHP 8.2+** (`composer.json`). Enable extensions used by Laravel: `openssl`, `pdo_mysql`, `mbstring`, `tokenizer`, `xml`, `ctype`, `json`, `fileinfo`, `bcmath` (if used).

3. **Upload codebase**

   Upload the **`backend/`** tree (composer files, `app/`, `bootstrap/`, `config/`, `database/`, `public/`, `routes/`, `storage/`, `.env`).  
   Omit frontend `node_modules`; optional omit `backend/tests`.

4. **Composer**

   On host SSH or panel Composer UI:

   ```bash
   cd /path/to/backend
   composer install --no-dev --optimize-autoloader
   ```

5. **Environment**

   ```bash
   cp .env.production.example .env
   ```

   Edit `.env`:

   | Variable | Purpose |
   |----------|--------|
   | `APP_ENV=production` | Production mode |
   | `APP_DEBUG=false` | No stack traces to clients |
   | `APP_KEY` | Run `php artisan key:generate` once |
   | `APP_URL` | **`https://` + canonical API hostname** |
   | `APP_FORCE_HTTPS=true` | Force `https:` in URLs generated by Laravel |
   | `TRUST_ALL_PROXIES=true` | Trust `X-Forwarded-Proto`/Host behind panel proxy (HTTPS detection on many hosts) |
   | `DB_*` | MySQL from DirectAdmin (“MySQL Databases”) |
   | `CACHE_STORE=file` _(and optionally `CACHE_DRIVER=file`)_ | Laravel 11 honours **`CACHE_STORE`**; **`file`** avoids DB cache migrations on install. |
   | `SANCTUM_STATEFUL_DOMAINS` | **Your web + storefront hosts** (+ Capacitor defaults if SPA cookies are used) |
   | `CORS_ALLOWED_ORIGINS` | **Comma list** of exact HTTPS origins of the SPA / Capacitor shell (`Origin` header); for `diigiimall.ir` plus Capacitor, see **`backend/.env.production.example`** |

6. **Migrations**

   ```bash
   php artisan migrate --force
   ```

   Optional demo data (once): `php artisan db:seed --force`

7. **Permissions**

   ```bash
   chmod -R 775 storage bootstrap/cache
   ```

   Ownership (`www/apache/nobody`) per hosting docs — often correct user + web group.

8. **Optimization (after `.env` is stable)**

   ```bash
   php artisan config:cache
   php artisan route:cache
   ```

   After env/routes/config changes:

   ```bash
   php artisan optimize:clear
   php artisan config:cache && php artisan route:cache
   ```

9. **Public storage URLs**

   If product images land in `storage/app/public`:

   ```bash
   php artisan storage:link
   ```

10. **Smoke tests**

   - `curl -sS https://your-api-domain/api/health` → JSON `status: success`
   - OPTIONS preflight (`Origin: https://localhost`, `Access-Control-Request-Method: POST` to `/api/auth/login`) → `Access-Control-Allow-Origin` echoes allowed origin(s).

## Required server configuration

| Requirement | DirectAdmin expectation |
|-------------|-------------------------|
| **Web root** | `public/index.php` is the sole front controller (not `_project_root/backend`). |
| **HTTPS** | Valid certificate matching `APP_URL`. |
| **TLS termination + proxy** | `TRUST_ALL_PROXIES=true` + `trustProxies(at: '*')` so Laravel honours `X-Forwarded-Proto`. |
| **PHP** | 8.2+; **`memory_limit` ≥ 256M** for CLI Composer if helpful (MultiPHP / `.user.ini`). |
| **Database** | MySQL/MariaDB user bound to `%`/`localhost`, matching `.env`. |
| **Rewrite** | `mod_rewrite` or LiteSpeed equivalent; Laravel `public/.htaccess` rewrites **`HTTP_AUTHORIZATION`**. |

### Document root not on `public/` (last resort)

Moving `public/*` upward and patching `index.php` paths toward `bootstrap/` works but is brittle — prefer subdomain docroot = `backend/public`.

### Do not expose

Keep `.env`, `storage/logs`, vendor outside executable docroot semantics; web layer should serve only **`public/`**.

## CORS (Capacitor Android)

Capacitor with **`androidScheme: https`** typically sends **`Origin: https://localhost`**. Include in **`CORS_ALLOWED_ORIGINS`**:

- `https://localhost`
- `capacitor://localhost` (legacy / tooling)

Plus every storefront origin (`https://www.yoursite.ir`, CDN preview URLs, staging). Explicit origins pair best with **`CORS_SUPPORTS_CREDENTIALS=true`**; use **`false`** if you only ship Bearer tokens and never use cookie credentials on fetch — see **`config/cors.php`**.

---

## Sanctum token authentication

- Clients use **`Authorization: Bearer <sanctum_token>`**. Routes use **`auth:sanctum`** (`routes/api.php`). No cookies required for the mobile Bearer flow.
- **`SANCTUM_STATEFUL_DOMAINS`** restricts **Stateful Sanctum** (SPA sessions). Token APIs still rotate/revoke via `personal_access_tokens`. Empty domains are sanitized in **`config/sanctum.php`**.

---

## HTTPS production support

1. **`APP_URL=https://...`** canonical host.
2. **`APP_FORCE_HTTPS`** + **`AppServiceProvider`** call **`URL::forceScheme('https')`** where appropriate.
3. **`TRUST_ALL_PROXIES`** in `.env` → **`bootstrap/app.php`** calls **`trustProxies(at: '*')`** — required on many proxies so `secure()`/`url()` match client HTTPS.

## Production readiness verification (DigiMall `backend`)

### Production PHP / Composer dependencies

| Item | Status |
|------|--------|
| **Runtime** | `composer.json` requires **PHP ≥ 8.2** (`^8.2`). Hosting MultiPHP must match. |
| **Production packages** (`require`) | `laravel/framework`, `laravel/sanctum`, `laravel/tinker`. |
| **`require-dev`** | Faker, Pint, Mockery, Collision, PHPUnit, Ignition — **excluded** with `composer install --no-dev`. |
| **Lock file** | Commit and deploy **`composer.lock`** so installs are reproducible. |
| **`config.platform`** | Not pinned in this repo; if hosting PHP is older, Composer will fail loudly (good). |
| **`laravel/tinker`** | Ships in **`require`** (prod); optional hardening: `composer remove laravel/tinker` if you forbid REPL tooling on prod. |

### Optimized autoload & performance

- **`composer.json`** already sets **`"optimize-autoloader": true`**. With **`composer install --no-dev`**, Composer builds an optimized **`vendor/composer`** class map.
- Optional stricter prod autoload (**larger regenerations on each deploy**):

  ```bash
  composer dump-autoload --optimize --classmap-authoritative --no-dev
  ```

- **Laravel optimizations** on the server (after `.env` is correct): **`config:cache`**, **`route:cache`**. This app **has no `resources/views`**, so skip **`view:cache`** unless you add Blade views (`view:cache` can error or provide no gain).
- Optional: **`php artisan event:cache`** if you introduce many event/listener classes and measure a win.

### Storage symlink (`public/storage`)

- **`config/filesystems.php`** declares the default link **`public/storage` → `storage/app/public`** (`'links'` key).
- After deploy:

  ```bash
  php artisan storage:link
  ```

  **Compatibility:** DirectAdmin/OpenLiteSpeed may block symlinks (“FollowSymLinks”). If **`storage:link`** fails or HTTP 403s **`/storage/...`**:

  - Reconfigure the vhost/docroot permission model, **or**
  - Use a hosting-specific “extra MIME / alias” workaround, **or**
  - Serve uploaded assets from a subdomain with symlink allowed (provider-dependent).

See also **`FILESYSTEM_DISK`** and **`APP_URL`** (`public` disk `url` = `APP_URL/storage`).

### Writable paths (ownership + chmod)

Directories the **PHP user running the web/CLI context** must be able **create/write** under:

| Path | Why |
|------|-----|
| `storage/framework/cache/**` | framework file cache (`CACHE_DRIVER=file`) |
| `storage/framework/sessions` | if `SESSION_DRIVER=file` ever used via web middleware |
| `storage/framework/views` | compiled Blade (`php artisan view:clear` clears) |
| `storage/logs` | `single`/`stack` log channel |
| `storage/app/` (and **`storage/app/public`** if uploads) | local disk storage |
| `bootstrap/cache/` | **`config.php`**, **`routes-*.php`**, **`services.php`**, package caches |

Recommended one-liner **on the server**:

```bash
chmod -R u+rwX,go+rX bootstrap/cache storage
```

Use **`775`** + **`chown`** to **`user:apache|www-data|nobody`** per host docs (**avoid global `chmod 777`**).

### Laravel optimization commands (compatibility)

| Command | Compatible with this codebase? |
|---------|---------------------------------|
| `php artisan config:cache` | **Yes.** Run after `.env` is production-final. Changing `.env` later requires **`config:clear`** or **`optimize:clear`**. |
| `php artisan route:cache` | **Yes** (after fix below): API routes use controller actions / invokables only; Laravel’s stock **`health: '/up'` closure is not used.** |
| `php artisan optimize` / `optimize:clear` | **Yes.** `optimize` runs the framework’s curated cache steps (`optimize` = config + routes + … per Laravel version). Prefer **`optimize:clear`** before rebuilding caches during deploys. |

### Route caching compatibility

- **Previously:** Laravel 11 registers **`GET /up`** as a **closure**, which **`route:cache` cannot serialize**.
- **This project:** **`bootstrap/app.php`** registers **`Route::get('/up', …SystemHealthController::class)`** and calls **`PreventRequestsDuringMaintenance::except('/up')`** so maintenance mode parity is preserved **without** a closure route.
- **`routes/api.php`:** **`/health`** and **`Route::fallback`** use invokable controllers — OK for **`route:cache`**.

Sanity-check **before deployment**:

```bash
php artisan route:cache
```

If anything fails with “unable to serialize…”, **`route:clear`** and fix remaining closure routes before going live.

### `APP_KEY` handling

- **`APP_KEY` must be unique per environment** (`base64:`…). Laravel encrypts cookie/session payloads and uses it for encryption helpers.
- **Do not ship your local `.env` with production secrets.** Create production `.env` on the server (`cp .env.production.example .env`) then:

  ```bash
  php artisan key:generate --force
  ```

  **`--force`** overwrites missing/placeholder keys (use cautiously once you know migrations aren’t sealing old encrypted data—in this starter app that’s unlikely).
- **`config:cache` bakes config** from the `.env` **present when you run cache**; rotate keys safely with Laravel’s **`APP_PREVIOUS_KEYS`** if you encrypt user data across rotation (mostly N/A unless you leverage `encrypt()` for stored secrets).

## Deployment blockers (known)

| Blocker | Mitigation |
|---------|-------------|
| **PHP `< 8.2` or missing PDO/MySQL/ext** | Fix MultiPHP + extensions (`pdo_mysql`, `openssl`, `mbstring`, …). |
| **Document root ≠ `backend/public`** | Point subdomain docroot there or follow host doc for nested installs. |
| **`route:cache` + Laravel default `/up` closure** | **Resolved** in-repo via **`SystemHealthController`**. |
| **Symlinks disabled (`storage:link`)** | See “Storage symlink” above. |
| **Non-writable `storage`/`bootstrap/cache`** | 500s and failed deploy scripts; chmod/chown accordingly. |
| **Running Composer on host with low `memory_limit`** | Use local `composer install --no-dev` + upload **`vendor/`**, or raise CLI `memory_limit` in `.user.ini`/`php.ini`. |
| **Platform functions disabled** (`proc_open`) | Composer / artisan may misbehave; enable for deploy user or vendor-upload workflow. |

## 1. Commands to run **locally** before upload

From the **`backend/`** directory (POSIX shell):

```bash
cd backend

composer validate --no-check-publish

composer install --no-dev --optimize-autoloader

# Optional: tighter autoload (skip if incremental deploy churn is painful).
composer dump-autoload --optimize --classmap-authoritative --no-dev

# Prove HTTP routes serialize (temporary dummy key avoids “No application encryption key” mid-command).
export APP_ENV=production
export APP_KEY="base64:$(openssl rand -base64 32)"
php artisan route:cache
php artisan optimize:clear
unset APP_ENV APP_KEY

# Produce a tarball/zip omitting bulky dev-only dirs (adjust paths for your tooling).
# From monorepo root (parent of `backend/`):

git archive --format=zip HEAD:backend -o digimall-api.zip
```

Notes:

- **Final** `config:cache`/`route:cache` should normally run **on the production server after real `.env` + `APP_KEY`** are in place (`php artisan optimize:clear && php artisan config:cache && php artisan route:cache`).
- **`database/database.sqlite`** may exist locally; **omit** SQLite artifacts from uploads if prod uses **`DB_CONNECTION=mysql`**.

Windows PowerShell equivalent for **`APP_KEY`** + route check:

```powershell
cd backend
composer install --no-dev --optimize-autoloader
$env:APP_ENV = 'production'
$bytes = New-Object byte[] 32
[Security.Cryptography.RandomNumberGenerator]::Create().GetBytes($bytes)
$env:APP_KEY = 'base64:' + [Convert]::ToBase64String($bytes)
php artisan route:cache
php artisan optimize:clear
Remove-Item Env:APP_ENV, Env:APP_KEY
```

## 2. Exact files / folders **to upload** (`backend`)

Upload the **whole Laravel tree** under **`backend/`**, excluding dev noise / machine-local rubbish:

### Always include

- `app/`, `bootstrap/`, `config/`, `database/`, `public/`, `routes/`, `composer.json`, `composer.lock`

### Include **either** local `vendor/` **or** plan to **`composer install` on server**

- **`vendor/`** — upload **if hosting has no Composer/SSH**.
- Omit **`vendor/`** if SSH + Composer available (lighter upload).

### Environment templates (recommended)

- **`.env.production.example`** (template).  
- **`DEPLOYMENT_DIRECTADMIN.md`** (ops doc).  

### Do **not** upload from dev machines unless intentional

| Exclude | Reason |
|---------|--------|
| **`.env`** (committed secrets) | Create fresh `.env` on server (`cp .env.production.example .env`). Never leak credentials. |
| **`tests/`** | Optional omission (smaller footprint). Production doesn’t execute them unless you purposely run PHPUnit in CI. |
| **`node_modules/`** | Frontend lives outside `backend/`; should not exist here. |
| **`.git/`** | Usually omitted from vendor zip uploads (not harmful if present). |
| **Dev SQLite `database/database.sqlite`** | Omit if production uses MySQL; prevents accidental SQLITE usage. |

### After unpacking on server before going live

1. **`cp .env.production.example .env`** → edit `APP_URL`, DB, mail, queues, HTTPS/proxy vars.  
2. **`composer install --no-dev --optimize-autoloader`** (unless `vendor/` uploaded).  
3. **`php artisan key:generate`**.  
4. **`chmod` writable paths** + **`php artisan migrate --force`**.  
5. **`php artisan storage:link`**.  
6. **`php artisan optimize:clear`**, then **`php artisan config:cache`**, **`php artisan route:cache`**.

## 4. Shared-hosting compatibility warnings

1. **Document root mismatch** — public URLs must resolve to **`backend/public`** ( **`index.php` front controller ).  
2. **Symlink bans** → **`storage:link` / `/storage`** failures; see symlink section.
3. **TLS termination mismatch** (`http` internally vs `https` externally) — use **`TRUST_ALL_PROXIES`** + **`APP_FORCE_HTTPS`** as documented earlier.
4. **Disabled PHP functions / low memory** hurts Composer CLI and large migrations.
5. **`open_basedir`** / jail may block unexpected paths (`storage/logs` outside jail).
6. **Cron gaps** — `QUEUE_CONNECTION=sync` is fine for MVP; swapping to **`database/redis`** queues later requires **`php artisan queue:work`** under **cron/supervisor** (often unavailable/costly on bottom-tier shared plans).
7. **`laravel schedule`** — **`routes/console.php` exists** only for custom Artisan defs (`db:ensure-created`, `inspire`); **`php artisan schedule:run`** is optional until you attach scheduled jobs in **`bootstrap/app.php` → `withSchedule`**.
8. **`mod_security`/`secrules`** blocking JSON bodies (`POST /api/auth/login`) — whitelist API route if uploads fail sporadically.
